Project 6

CRUD Web App in PHP

Software

In this project, I developed a PHP web application to manage project records in a PostgreSQL database. First, I created a table named 'proyectos' with fields for ID, name, description, start date, end date, and status. Then, I set up the main files: 'index.php' to display the project list, 'crear.php' to add new projects, 'editar.php' to update project details, and 'eliminar.php' to delete records. I implemented SQL queries for each CRUD operation, ensuring data integrity and error handling. Additionally, I added a status dropdown with options like "Planned," "In Progress," and "Completed" to keep project statuses organized. Finally, I tested each feature to confirm the correct functioning of the app, focusing on data updates and record deletion.

⟡ Reflection ⟡

1. What difficulties or obstacles did you experience while working on this project?

Setting up the PostgreSQL connection and ensuring all CRUD operations worked correctly was challenging, especially handling errors during the editing and deleting processes.

2. What skills did you apply during this project?

I applied my knowledge of PHP to handle form data, SQL to manage the database, and basic HTML/CSS to structure and style the interface.

3. Did you develop or improve any new skills? If so, how?

Yes, I improved my ability to work with PostgreSQL and learned how to implement dropdown menus in PHP using the 'select' element.

4. How could you improve this project if given more time or resources?

I would add input validation to prevent SQL injection and implement a search function to filter projects based on their status or date range.
